# 文档与知识归档

## 语言

项目文档、README、设计、实施计划、验证报告、Trellis 任务材料、`.agents` Skill/参考资料和 `.codex` Agent/提示词默认使用简体中文。命令、代码标识符、文件名及没有通行中文译名的专有名词可保留原文。

Git 提交主题和正文默认使用简体中文；Conventional Commits 的 `type(scope):` 可作为机器可识别前缀保留英文标识符。

## 内容归属

| 内容 | 位置 |
|---|---|
| 每轮必读安全红线 | `AGENTS.md` |
| 长期产品/代码/测试/发布规则 | `.trellis/spec/<domain>/` |
| 当前任务需求、设计、计划、进度和证据 | `.trellis/tasks/<task>/` |
| 研究资料 | 当前任务 `research/` |
| 个人跨会话摘要 | `.trellis/workspace/<developer>/` |
| 过期运行结果和已完成一次性计划 | Git 历史，不保留活动副本 |

## 索引规则

每个领域 `index.md` 只提供用途、开发前检查、质量检查和具体文件链接。任务只选择相关规范，禁止把整个 `.trellis/spec/` 全量注入每轮上下文。Trellis 自带 `guides/` 保留为短思考清单，不复制项目具体契约。

## 翻译与生成内容

- 翻译由脚本生成的 Markdown 标题、区块、默认文字或 JSONL `_example` 种子时，必须同时修改生成器，不能只修改当前产物。JSONL 的字段名、占位符和命令字面量保持原样。
- 迁移期读取器应兼容已有英文标签；稳定生成行为必须有持久回归测试，使用 `TemporaryDirectory` 或 mocked Git 覆盖首次生成、连续更新和旧标签读取，并接入项目标准检查。
- fenced block 中的可执行命令、路径、配置键、占位符和协议字面量保持原样；目录树注释、输出模板等面向人的内容可以翻译。确需纠正命令时，必须以本机目标版本的 `--help` 或实现为证据。
- 运行时文档批量修改后，搜索入口 Skill、参考资料和工作流中的同一命令/事件名，避免入口摘要继续传播已在详细参考中纠正的旧行为。
- 解析器依赖的标题、tag 块和兼容锚点按精确字面量保留；修改后运行对应解析入口，而不是只检查 Markdown 外观。

## 迁移与删除门禁

- 先读取完整源文档，逐条提取长期有效内容。
- 目标 spec 已写入、索引存在且安全映射检查通过后才能删除源文件。
- 已被新决定取代的设计、已完成实施计划和过期验证结果只保留 Git 历史。
- README 面向用户和开发者，不承载完整内部规则；进一步阅读链接指向 spec 索引。

## 更新要求

任务产生新稳定契约、易错点或调试结论时，同一任务内更新对应 spec。纯一次性命令输出只写任务检查点；不能为了“有文档”复制长期无价值的日志。
