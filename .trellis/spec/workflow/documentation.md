# 文档与知识归档

## 语言

项目文档、README、设计、实施计划、验证报告和 Trellis 任务材料默认使用简体中文。命令、代码标识符、文件名及没有通行中文译名的专有名词可保留原文。

## 内容归属

| 内容 | 位置 |
|---|---|
| 每轮必读安全红线 | `AGENTS.md` |
| 长期产品/代码/测试/发布规则 | `.trellis/spec/<domain>/` |
| 当前任务需求、设计、计划、进度和证据 | `.trellis/tasks/<task>/` |
| 研究资料 | 当前任务 `research/` |
| 个人跨会话摘要 | `.trellis/workspace/<developer>/` |
| 过期运行结果和已完成一次性计划 | Git 历史，不保留活动副本 |

## 索引规则

每个领域 `index.md` 只提供用途、开发前检查、质量检查和具体文件链接。任务只选择相关规范，禁止把整个 `.trellis/spec/` 全量注入每轮上下文。Trellis 自带 `guides/` 保留为短思考清单，不复制项目具体契约。

## 迁移与删除门禁

- 先读取完整源文档，逐条提取长期有效内容。
- 目标 spec 已写入、索引存在且安全映射检查通过后才能删除源文件。
- 已被新决定取代的设计、已完成实施计划和过期验证结果只保留 Git 历史。
- README 面向用户和开发者，不承载完整内部规则；进一步阅读链接指向 spec 索引。

## 更新要求

任务产生新稳定契约、易错点或调试结论时，同一任务内更新对应 spec。纯一次性命令输出只写任务检查点；不能为了“有文档”复制长期无价值的日志。
