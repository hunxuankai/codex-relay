# Windows 代码签名

## 当前状态

仓库不包含代码签名证书或签名秘密。自行构建的 `CodexRelay.exe` 和 NSIS 安装器可能被 SmartScreen 显示为“未知发布者”。未签名产物不得描述为已签名或可信发布者。

## 正式发布流程

1. 在干净、隔离的 Windows 环境运行完整检查和 Release/NSIS 构建。
2. 扫描仓库、构建日志和待签名产物流程中的秘密。
3. 使用获授权的可信 Windows 代码签名证书签署主程序和安装器。
4. 使用 Windows 签名工具验证签名链、发布者和时间戳。
5. 记录实际 SHA-256、文件路径、签名验证输出和证书非秘密元数据。

## 秘密边界

- 证书私钥、密码、令牌和签名服务凭据不得提交到 Git 或写入 Trellis 资料。
- 没有证书和明确签名授权时，只能完成未签名构建和说明后续步骤。
- CI 日志必须避免打印凭据，签名失败必须保留真实错误，不以未签名产物替代成功。

## 验收区分

| 状态 | 可声明内容 |
|---|---|
| `npm run build` 通过 | Release/NSIS 已生成，仍可能未签名 |
| 签名命令通过 | 产物已执行签名步骤 |
| 独立验证链和时间戳通过 | 可以声明实际签名验证成功 |
| 未执行安装 | 不能声明安装、升级或卸载行为成功 |
