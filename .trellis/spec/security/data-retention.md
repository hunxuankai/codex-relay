# 数据保留与清理

## 数据敏感级别

`providers.json`、`auth.json` 和备份中的原始快照均可能包含明文 API Key，敏感级别相同。`metadata.json` 不得包含密钥，只记录事务、时间、操作、Provider ID、文件存在状态和应用版本。

## 备份保留

- 每次受管写操作前创建事务备份。
- 手动恢复前再备份当前状态。
- 最多保留最近 20 份，清理时不得删除当前事务仍依赖的备份。
- 备份页只展示元数据和恢复操作，不显示快照密钥。

## 卸载契约

NSIS 卸载器不得自定义删除：

- Codex 配置目录；
- `%LOCALAPPDATA%\CodexRelay`；
- API Key、日志、备份或开发者的其他数据。

该策略避免擅自造成不可恢复的数据损失，也意味着用户必须在确认不再需要恢复后自行安全清理。

## 损坏与事件响应

- 损坏 `providers.json` 的保留副本也可能含有效密钥，应提示用户重新设置并评估泄漏风险。
- 怀疑泄漏时：退出应用、在 Provider 平台吊销并轮换密钥，检查当前认证、Provider 存储、备份、日志、开发目录和同步软件。
- 从界面清空密钥只改变本地状态，不会吊销远端凭据。

## 禁止模式

- 不因卸载、升级、重装或“清理无用文件”自动删除用户配置和备份。
- 不把备份元数据“不含密钥”误写成“备份不含密钥”。
- 不在未验证实际清理范围时声称数据已彻底删除。
