# 可访问性与界面行为

## 语义与键盘

- 交互使用原生 `button`、`input`、`label`、导航和对话框语义，不用仅带 click 的 `div`。
- 所有按钮和表单控件可通过键盘到达，焦点态清晰可见。
- 图标按钮提供可理解的 `aria-label` 或可见文本。
- 对话框支持明确标题、取消/确认动作和合理的焦点管理。

## 状态表达

- 当前 Provider、无密钥、无效配置、warning/error 不能只靠颜色区分。
- 禁用操作仍要说明原因，例如当前 Provider 禁止删除、缺少密钥禁止启用。
- 成功切换同时提供应用内提示与 Windows 通知，并包含“请重启 Codex 后生效”。
- 备份页不得显示备份快照中的明文密钥。

## 布局与主题

- 保持约 900×620 的可缩放双栏布局，并在窄窗口下可读可操作。
- 颜色使用 CSS 变量跟随系统明暗主题；文本与控件保持足够对比度。
- 避免复杂动画和会阻碍操作的过渡；系统托盘关闭语义要和真正退出明确区分。

## 测试点

组件测试至少覆盖标签关联、按钮可用/禁用原因、焦点可见相关 CSS 契约、错误/成功文本和窄窗口关键布局类；真实托盘和 Windows 窗口行为需要人工或系统级证据，不能由单元测试代替。
