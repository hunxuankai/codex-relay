# 工作区索引

> 记录所有开发者与 AI Agent 协作的工作记录

---

## 概览

本目录跟踪项目中所有使用 AI Agent 的开发者记录。

### 文件结构

```
workspace/
|-- index.md              # 本文件，主索引
+-- {developer}/          # 每位开发者的目录
    |-- index.md          # 包含会话历史的个人索引
    |-- tasks/            # 任务文件
    |   |-- *.json        # 活动任务
    |   +-- archive/      # 按月归档的任务
    +-- journal-N.md      # 按 1、2、3... 排序的日志文件
```

## 开始使用

### 新开发者

运行初始化脚本：

```bash
python ./.trellis/scripts/init_developer.py <your-name>
```

该脚本会：
1. 创建被 Git 忽略的身份文件
2. 创建进度目录
3. 创建个人索引
4. 创建首个日志文件

### 已有开发者

1. 获取开发者名称：
   ```bash
   python ./.trellis/scripts/get_developer.py
   ```

2. 读取个人索引：
   ```bash
   cat .trellis/workspace/$(python ./.trellis/scripts/get_developer.py)/index.md
   ```

## 规则

### 日志文件规则

- 每个日志文件**最多 2000 行**
- 达到上限时创建 `journal-{N+1}.md`
- 创建新文件时更新个人 `index.md`

### 会话记录格式

每个会话应包含：
- 摘要：单行说明
- 分支：工作所在分支
- 主要改动：修改了什么
- Git 提交：提交哈希和消息
- 下一步：后续要做什么

---

## 会话模板

记录会话时使用以下模板：

```markdown
## 会话 {N}：{标题}

**日期**：YYYY-MM-DD
**任务**：{task-name}
**分支**：`{branch-name}`

### 摘要

{单行摘要}

### 主要改动

- {改动 1}
- {改动 2}

### Git 提交

| 哈希 | 消息 |
|------|---------|
| `abc1234` | {提交消息} |

### 测试

- [OK] {测试结果}

### 状态

[OK] **已完成** / # **进行中** / [P] **受阻**

### 下一步

- {下一步 1}
- {下一步 2}
```
