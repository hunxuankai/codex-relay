# 配置事务、验证与回滚

## 1. 受管文件

事务可能触及：

- Codex `config.toml`
- Codex `auth.json`
- Codex Relay `providers.json`
- 应用数据目录中的事务标记与备份

所有 Provider 创建、编辑、删除、切换、同步和恢复都必须经过 `TransactionService`。不得在命令、托盘或 Vue 层直接写这些文件。

## 2. 事务前读取与冲突检测

前端编辑会话携带三个文件的 `exists`、长度、修改时间和 SHA-256 指纹。服务获取全局异步写锁后重新读取最新状态，并计算当前指纹。

若请求指纹与当前指纹不同，返回外部修改冲突。事务不会把旧编辑结果覆盖到外部新内容。即使初次指纹一致，创建备份后、正式写入前还会再次比较，缩小竞争窗口。

## 3. 内存生成

- `config.toml` 使用 `toml_edit::DocumentMut` 局部修改。
- `auth.json` 使用固定两空格缩进、终止换行的 JSON。
- `providers.json` 保留版本与其他 Provider，只修改目标密钥。
- 在任何正式文件替换前，所有目标内容都先在内存生成并通过基础校验。

未知 TOML 字段、注释、其他 Provider 和顶层功能配置不得丢失。目标 Provider 没有默认模型时，不删除或覆盖原顶层 `model`。

## 4. 快照与备份

事务开始状态记录每个受管文件“存在或不存在”以及存在时的原始字节。备份目录以时间和事务 ID 命名，包含存在的原文件快照与 `metadata.json`。

元数据只记录事务、时间、操作、Provider ID、文件存在状态和应用版本，不保存 API Key。但是 `auth.json`、`providers.json` 快照本身可能含明文密钥，因此整个备份目录属于敏感数据。

## 5. 临时文件、验证与替换

每个写入按以下顺序执行：

1. 在目标同目录创建唯一临时文件。
2. 写入全部字节并刷新缓冲区。
3. 按文件类型解析临时内容：TOML、认证 JSON、Provider 密钥 JSON 或事务标记 JSON。
4. 替换正式文件。
5. 重新读取正式文件并验证业务不变量。

同目录临时文件避免跨卷替换问题。临时内容解析失败时不替换正式文件。

## 6. 写后业务验证

不同操作提供不同验证器：

- 创建/编辑：目标 Provider 字段与预期一致，密钥动作正确。
- 切换/同步：顶层当前 Provider、模型规则和 `auth.json` 密钥一致。
- 删除：目标 Provider 与目标密钥已删除，其他数据保留。
- 恢复：文件字节和原存在状态与备份快照一致。

事务成功后移除事务标记并返回新的文件指纹。

## 7. 回滚

任一阶段失败时，事务按开始快照恢复所有已触及文件：

- 原先存在：写回原始字节。
- 原先不存在：删除事务新建的正式文件。

回滚写入不复用前向业务验证，而是以精确恢复原始状态为目标。恢复后重新读取并比较快照。

只有所有触及文件都恢复且验证成功，公开错误才能声明“原配置已恢复”。若回滚写入、删除或验证任一步骤失败，返回 `TRANSACTION_ROLLBACK_INCOMPLETE`，保留事务标记并提示用户立即从备份页恢复。不得为了安抚用户而虚假显示“未造成更改”。

## 8. 恢复备份

手动恢复也是事务：

1. 验证备份目录名，拒绝路径穿越。
2. 读取并验证元数据及所需快照文件。
3. 获取全局写锁。
4. 先备份当前状态。
5. 写回备份字节或按“不存在”状态删除文件。
6. 重新解析和验证。
7. 刷新 Provider、托盘和扩展自检。

若恢复已完成但后续 Provider/自检刷新失败，界面会提示“状态刷新未完全成功”，不会把两个阶段混为一个成功。

## 9. 并发与文件监控

所有 `TransactionService` 克隆共享同一个 Tokio 互斥锁。应用写入期间，`AppState` 安装文件监控守卫；完成后登记最终指纹，只抑制匹配自身事务的事件。并发托盘/界面切换被忙锁和事务锁串行化。

## 10. 测试证据

单元测试注入 `FileOps` 故障，覆盖临时 TOML/JSON 无效、config/auth 写失败、替换后验证失败、回滚失败、并发事务和备份恢复。集成测试使用临时根运行 create→edit→switch→delete→restore，并验证注释、未知配置、认证密钥与原始字节。

`path_safety` 额外把默认 `USERPROFILE`/`LOCALAPPDATA` 指向带损坏文件的安全哨兵目录；显式覆盖工作流必须成功且哨兵目录递归快照完全不变。
